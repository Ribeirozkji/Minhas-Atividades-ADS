## Visão Geral

O **WEB PASS** é uma aplicação web front-end desenvolvida para cadastro e organização de informações de usuários e localidades.

O projeto simula um cenário real onde o briefing original foi perdido, exigindo a reconstrução completa dos requisitos antes da implementação.

---

## Problema

A falta de um briefing estruturado compromete o desenvolvimento de sistemas, gerando falhas de entendimento e retrabalho.

---

## Solução

- Reconstrução do briefing (preliminar + completo)  
- Definição clara de requisitos  
- Implementação de sistema funcional  
- Armazenamento de dados no navegador  

---

# BRIEFING DO PROJETO

## 1. Briefing Preliminar

**Público-alvo:**  
Usuários que precisam organizar informações de pessoas e localidades (estudantes, uso pessoal).

**Objetivos do sistema:**  
- Cadastrar usuários  
- Cadastrar localidades  
- Organizar dados  
- Validar informações  

**Imagem do sistema:**  
Simples, funcional, moderno e confiável  

**Conteúdo dinâmico:**  
Sim (dados inseridos pelo usuário)

**Concorrência:**  
- Excel  
- Google Sheets  
- Sistemas CRUD simples  

**Diferencial:**  
Interface mais intuitiva e uso direto no navegador  

**Observações:**  
Sistema 100% front-end utilizando localStorage
